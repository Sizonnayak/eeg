================================================================================
  TROUBLESHOOTING
================================================================================

ERROR: "MPS backend out of memory"
----------------------------------
Reduce batch size in train_loso_optimized.py:
  - Find: self.batch_size = 256
  - Change to: self.batch_size = 128


Training slower than expected (> 10 min)
-----------------------------------------
Normal for 50 epochs, but check:
  - "Using device: mps" appears at start
  - GPU usage: run `top -l 1 | grep GPU` in another terminal


LR scheduler not reducing LR
-----------------------------
Check patience setting:
  - Default: patience=10 (waits 10 epochs before reducing)
  - If loss decreases every epoch, LR won't reduce (good sign!)


Results same as baseline (65.1%)
---------------------------------
Possible causes:
  - Model needs more than 50 epochs (try 75)
  - F1=16 needs higher LR (try 0.003)
  - Check if F1 actually changed in config
